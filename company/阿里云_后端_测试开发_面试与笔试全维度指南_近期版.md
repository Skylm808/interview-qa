## 一、 最新阿里云测评 (Online Assessment) 情报

- **测评现状**:
  
  - 近 60 天内，我**没有检索到足够多的阿里云测评一手贴**，因此不能高置信地下结论说“阿里云已经换了新测评平台”或者“依旧完全等于北森题库”。
  - 近期更容易看到的是流程类反馈：有同学**笔试前就先收到面试邀约**，也有人处在“笔试/测评/面试”节奏错位的状态。这说明阿里云不同团队的流程推进并不完全统一。
  - **目前没有近两个月的公开强证据能证明阿里云测评启用了强制摄像头监考或严格切屏限制。**

- **题型拆解**:
  
  - 由于近 60 天缺乏稳定的一手测评题面，当前更稳妥的准备方式仍然是按“传统行测风格”兜底：
    - 图形推理
    - 数字推理 / 资料分析
    - 言语理解
    - 性格测试
  - 但要注意，**阿里云近两个月公开重点已经明显转向笔试和技术面**，所以不要把主要精力都押在测评上。

- **近期考生建议**:

  - 如果页面风格确实像传统行测，优先练：
    - 图推速度

## 二、 近期阿里云笔试 (Written Exam) 题库与考点

### 1. 笔试整体概况 (聚焦近期春招/实习)

- 近两个月能公开复核到的样本，主要集中在 **2026-03-25 到 2026-03-31**。
- 当前能看到两类高频结构：
  - **AI/Agent 研发卷**：`20 道选择 + 3 道编程 + 1 道 Prompt 题`
  - **算法/AI 相邻卷**：基础单选、基础多选、`NLP 单选 2 题`、`NLP 多选 2 题`、`3 道编程题（50 分）`
- 2026-03-25 的牛客改编帖明确提到：**阿里近期多条业务线笔试题基本共用同一套公开机试，淘天、阿里云等方向都可参考这场。**
- 从近样本看，阿里云笔试的趋势不是“纯 Hot100 默写”，而是：
  - 客观题里混合 **CS 基础 + AI/Agent 基础**
  - 编程题偏 **业务包装后的数学 / 构造 / 博弈 DP / 离线处理 / 字符串与输入处理**
- **难度趋势**:
  - 不能只靠 3 道经典模板题准备。
  - 题面包装感更强，很多题第一眼不像力扣原题。
  - AI 方向卷里，**Prompt 题和 Agent 基础题已经开始出现**。

  - Redis 为什么能做分布式锁，以及常见失效场景

  - GC 机制
  - 类加载过程
  - 双亲委派
  - Spring Bean / 依赖注入
  - `ThreadLocal` 原理与问题

- **AI/Agent Infra 额外加问点**:

### 3. 编程题/算法题真实复盘

- **真题 1：双数组轮流选位置，求最终 `A-B`**
  
  - 题意：给两个数组 `a`、`b`，Alice 和 Bob 轮流选位置，Alice 选到位置 `i` 时对差值贡献 `a[i]-b[i]`，Bob 选到位置 `i` 时对差值贡献 `b[i]-a[i]`，问双方最优时最终 `A-B`。
  - 近期结论：这题关键不是博弈搜索，而是发现**无论谁选，差值贡献本质都能化成同一个量**，最后直接求和。
  - 对应题型：数学观察 / 博弈化简
  - 对应 LeetCode：**没有直接同题**，更像“先观察再一行化简”的比赛题。

- **真题 2：统计特殊完全平方数个数**
  
  - 题意：求 `n` 以内有多少个数，既是完全平方数，又能按十进制切成左右两段，且左右两段也都是完全平方数。
  - 近期结论：做法偏**数学枚举 + 判平方数**，不是传统 DP。
  - 对应题型：数论 / 枚举
  - 对应 LeetCode：**没有稳定映射**，属于阿里近期常见的“题意不复杂，但要先读懂条件”的题。

- **真题 3：对每个 `k`，求相邻差不超过 `k` 的最长子数组**
  
  - 题意：给数组 `a`，当 `k = 1, 2, ..., n` 时，分别求满足“相邻元素差值都不超过 `k`”的最长连续子数组长度。
  - 近期结论：公开思路是按相邻差从小到大处理，再不断合并区间，属于**离线处理 + 并查集 / 区间合并**。
  - 对应题型：离线算法 / 并查集
  - 对应 LeetCode：**无直接原题**，但思路和“按阈值逐步开放边、维护连通块大小”的套路接近。

- **真题 4：仓位配货表**
  
  - 题意：有多类货物，每类货物要分给 `k` 个仓位，同类货物在任意两个仓位上的数量差不能超过 1。问任选一个固定仓位，它最终拿到的总货物数最小值和最大值分别是多少。
  - 近期结论：按每类货物独立看，固定仓位拿到的是“基础份额”或“基础份额 + 1”，最后直接累加。
  - 对应题型：数学 / 贪心
  - 对应 LeetCode：**无直接同题**。

- **真题 5：两份排列的字典序最大公共子序列**
  
  - 题意：给两个排列，求它们的字典序最大的公共子序列。
  - 近期结论：这题不是普通 LCS，因为“排列”这个条件大幅简化了问题，重点在于**利用位置关系做贪心/构造**。
  - 对应题型：排列性质 + 贪心
  - 对应 LeetCode：**不建议硬套 LCS 题号**，这是典型的业务改编题。

- **你该怎么刷题**
  
  - 如果只剩 1 到 2 天，优先刷：
    - 字符串 / 输入处理
    - 构造题
    - 数学观察题
    - 树和图上的最长路径
    - 博弈 DP
    - 并查集离线题
  - **不要只刷 Hot100 经典中等题**，阿里云近题明显更爱“包装题面 + 看读题速度”。

## 三、 面试高频真题汇总 (后端 & 测开)

### 1. 语言基础与底层 (Go/Java等)

- Go 和 Java 的区别是什么
- Java `String` 为什么不可变
- GC 机制是什么，自动回收还是手动回收
- 类加载过程是什么，双亲委派是什么
- Spring Bean 是什么，生命周期怎么讲
- `ThreadLocal` 原理和常见坑
- 进程、线程、协程区别
- 项目里并发部分是怎么做的
- 手撕：
  - 无重复字符的最长子串
    - LeetCode `3. Longest Substring Without Repeating Characters`

### 2. 数据库与中间件 (MySQL, Redis, 消息队列等)

- MySQL 索引结构是什么
- 联合索引怎么命中
- Redis 怎么实现分布式锁
- 分布式锁再往下追问会问哪些边界场景
- gRPC 为什么适合服务间通信，为什么用二进制编码
- HNSW 是什么，向量检索怎么做
- `embedding` 和 `rerank` 的区别
- RAG 的分块策略怎么定
- 如果你项目里写了 MQ，近期虽然没有 Redis / MySQL 那么高频，但 Kafka / RabbitMQ 的场景对比仍然建议准备

### 3. 项目深挖与系统设计

- 面试官会持续拷打项目，不是只问八股

- 高频追问方向包括：
  
  - 你项目里某个模块的源码结构和核心数据结构是什么
  - 你为什么选 Higress，不选传统 Ingress
  - 为什么权限方案选 RBAC，不选 ABAC
  - 一个 Agent 有很多工具，但没匹配到想要的工具怎么办
  - MCP 的 `stdio` 和 `SSE` 有什么区别
  - MCP 和普通 REST API 有什么区别
  - Agent 调用 MCP 的完整链路是什么
  - 大模型流式输出是怎么实现的
  - 你平时怎么用 Cursor / Claude Code / AI Coding 工具
  - 如果让你现场做一次 AI Coding / Vibe Coding，会怎么落地

- **最近公开样本里一个很明显的新变化**:
  
  - 阿里云部分团队已经开始把 **AI Coding / 屏幕共享现场实现** 当成真实考察项，而不是只问口头八股。
  - 所以如果你投的是 Agent Infra、AI 应用、AIOps、SRE 这类团队，不能只会背八股，**要能把简历项目和实际代码链路讲清楚。**

### 4. 测试开发专属

- 近两个月阿里云测开的一手公开面经明显少于后端，但从最新职位和阿里系质量方向描述看，测开不是传统“点点点测试”，而是明显工程化：
  - 自动化测试框架设计
  - 压测平台
  - 故障演练
  - 代码审计
  - 架构分析
  - 全链路压测
  - AI 生成测试用例
  - 测试智能体
- 如果你面的是阿里云测开，建议额外准备：
  - 如何设计一个自动化测试平台
  - 如何做接口自动化和回归
  - 如何做流量回放和灰度验证
  - 如何设计故障演练场景
  - 如何衡量测试覆盖率与测试收益
  - 如何把 LLM / Agent 接到测试链路里
