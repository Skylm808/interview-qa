# Hive 面经整理：介绍、数仓定位与高频问题

> 面试导读：Hive 面试一般会问“它解决什么问题”“和 HBase / ClickHouse 怎么区分”“分区、分桶、Metastore 是什么”“为什么适合离线数仓”。下面先给一版简介，再整理常见问题。

## 1. Hive 是什么

Hive 是 Apache Hadoop 生态里的数据仓库软件，主要用 SQL 风格去读、写和管理分布式存储中的大数据。

它更适合：

- ETL
- 离线统计
- 报表分析
- 数仓分层加工

### 1.1 核心定位

- 不是在线事务数据库
- 更偏批处理和离线分析
- 常和 HDFS、Tez、MapReduce 等配套

### 1.2 常见能力

- SQL 查询
- UDF / UDAF / UDTF 扩展
- 数仓建模
- 大规模数据处理

## 2. Hive 面试高频问题

### 2.1 Hive 解决什么问题？

Hive 主要解决的是：

- 用 SQL 方式处理海量分布式数据
- 做离线 ETL 和数仓分析
- 让 Hadoop 生态的数据更容易被使用

### 2.2 Hive 和 HBase 有什么区别？

- Hive：离线数仓，偏批量计算
- HBase：在线随机读写，偏低延迟访问

可以直接记成：

> Hive 是“算”，HBase 是“查/写”。

### 2.3 Hive 和 ClickHouse 有什么区别？

- Hive：偏离线数仓，吞吐优先
- ClickHouse：偏在线分析，查询响应更快

如果面试官追问业务选型，可以答：

> 如果是大规模离线 ETL 和定时报表，优先考虑 Hive；如果是要给业务看板和交互式分析提供低延迟查询，ClickHouse 更合适。

### 2.4 什么是 Metastore？

Metastore 是 Hive 的元数据中心，保存表、字段、分区、位置等信息。  
面试中常问它的作用：

- 管理元数据
- 让 SQL 能找到真实数据文件
- 支持表和分区信息查询

### 2.5 什么是分区，为什么要分区？

Hive 分区通常用于减少扫描的数据量，比如按天、按月分区。

常见回答：

> 分区本质上是用目录层次帮助裁剪数据，避免全表扫描。

### 2.6 什么是分桶？

分桶是把数据按某个字段 hash 到固定数量的文件中，常用于：

- 提升 join 效率
- 提升采样效率
- 帮助数据管理

### 2.7 Hive 为什么适合离线而不适合低延迟查询？

因为 Hive 设计目标是批量处理和数仓分析，不是逐条事务式访问。  
实际查询链路通常会涉及扫描较多文件、调度计算任务，因此延迟一般高于在线分析数据库。

### 2.8 Hive 常用的执行引擎有哪些？

常见会提到：

- MapReduce
- Tez
- Spark

现代面试里通常会顺带问：

> 你的团队数仓是跑在 MR、Tez 还是 Spark 上？

### 2.9 Hive ACID 是什么？

Hive 后来逐步支持 ACID 特性，面试常会问它的事务能力、compaction、并发控制等问题。  
如果你只想答基础版，可以先说明：

> Hive 不是传统 OLTP 数据库，但它逐步补齐了一部分事务能力，主要还是服务于数仓场景。
